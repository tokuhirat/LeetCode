# 300. Longest Increasing Subsequence
## STEP1
- 何も見ずに解いてみる
- 手で解こうとすると意外と難しく、[1, 3, 5, 2, 3, 4, 5] の例を考えて、作れた部分列の中で最小の値を index ごとに持っていれば良いとわかり（左から見ていく過程で[1, 3, 5] -> [1, 2, 5] -> [1, 2, 3, 4]）、どこかでみたことがあるアルゴリズムを思い出した。
```python
import bisect


class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        if not nums:
            raise ValueError("nums is empty")
        
        # i-th element is the smallest number at i-th position of strictly increasing subsequence
        table = [nums[0]]
        for num in nums[1:]:
            index = bisect.bisect_left(table, num)
            if index >= len(table):
                table.append(num)
                continue
            table[index] = min(num, table[index])
        return len(table)
```

#### memo
時間計算量: $O(n\log n)$  
空間計算量: $O(n)$  
見直すと 0 番目を特別扱いする必要はない。

```python
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        # i-th value is max lis length using i-th num as last element
        table = []
        for num in nums:
            lis_length = 1
            for i, lis_length_so_far in enumerate(table):
                if nums[i] < num:
                    lis_length = max(lis_length, lis_length_so_far + 1)
            table.append(lis_length)
        return max(table)
```
